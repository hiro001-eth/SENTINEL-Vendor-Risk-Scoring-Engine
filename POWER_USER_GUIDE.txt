 PART 2 — INSTALLATION (Zero to Running in 5 Minutes)
================================================================================

STEP 1: Clone the repository
─────────────────────────────
  git clone https://github.com/yourorg/sentinel-vendor-risk-engine.git
  cd "sentinel-vendor-risk-engine"

STEP 2: Install all dependencies
──────────────────────────────────
  python3 -m pip install -e ".[dev,test]"

STEP 3: Copy and configure environment variables
──────────────────────────────────────────────────
  cp .env.example .env

  Open .env and set at minimum:
    QUESTIONNAIRE_SCHEMA_PATH=./rules/questionnaire_schema.yaml
    WEIGHT_CONFIG_PATH=./rules/weight_config.yaml
    THRESHOLD_CONFIG_PATH=./rules/threshold_config.yaml
    OUTPUT_DIR=./output

  Optional (unlock full power):
    BITSIGHT_API_KEY=your_key          → Enables live external ratings
    SECURITYSCORECARD_API_KEY=your_key → Enables live external ratings
    HIBP_API_KEY=your_key              → Enables breach intelligence
    OPENAI_API_KEY=your_key            → Enables AI PDF evidence verification
    WEBHOOK_URL=https://hooks.slack.com/... → Enables Slack/Teams alerts
    WEBHOOK_PROVIDER=slack             → or: teams, generic
    JIRA_URL=https://yourorg.atlassian.net
    JIRA_TOKEN=your_token              → Enables Jira ticket sync

STEP 4: Launch the platform
────────────────────────────
  OPTION A — CLI (for scoring a single batch):
    sentinel score ./data/sample_responses.csv

  OPTION B — Full REST API + Dashboard:
    sentinel api --host 127.0.0.1 --port 8000
    → Open browser: http://127.0.0.1:8000
    → API docs:     http://127.0.0.1:8000/docs

  OPTION C — Docker (One-command production deploy):
    docker compose up --build -d
    → API available at: http://localhost:8000

 PART 4 — BASIC USAGE: YOUR FIRST VENDOR ASSESSMENT
================================================================================

THE SCENARIO: You are a GRC analyst. A new vendor (Acme Cloud Corp) has just
been onboarded. You need to assess their security posture.

STEP 1: Register your analyst account
───────────────────────────────────────
  POST http://localhost:8000/auth/register
  Body (JSON):
  {
    "email": "analyst@yourcompany.com",
    "password": "SecureP@ss123",
    "role": "Analyst"
  }

  → You receive a session token. Save it.

STEP 2: Prepare the vendor questionnaire CSV
─────────────────────────────────────────────
  Create a file called acme_responses.csv with these columns:
    vendor_id, vendor_name, assessment_date, responded_by,
    Q_DH_01, Q_AC_01, Q_IR_01, Q_BC_01, Q_ENC_01

  Example row:
    V-ACME, Acme Cloud Corp, 2026-06-20, John Smith,
    yes, yes, no, partial, yes

  Column meanings:
    Q_DH_01  = Data Handling (Do you encrypt data at rest?)
    Q_AC_01  = Access Controls (Do you enforce MFA?)
    Q_IR_01  = Incident Response (Do you have an IR plan?)
    Q_BC_01  = Business Continuity (Do you have disaster recovery?)
    Q_ENC_01 = Encryption (Do you use TLS 1.2+ in transit?)

STEP 3: Submit the assessment
──────────────────────────────
  POST http://localhost:8000/assessments/
  Form-data: file = acme_responses.csv

  → Response includes a run_id like: SENTINEL-20260620-142300-A1B2C3D4
  → Save this run_id.

STEP 4: View the board-ready executive summary
───────────────────────────────────────────────
  GET http://localhost:8000/assessments/SENTINEL-20260620-142300-A1B2C3D4/executive-summary

  → Returns:
    - Overall posture: ACCEPTABLE / ELEVATED / CRITICAL
    - Financial exposure: $47,250 Annual Loss Expectancy
    - Top 5 remediation recommendations ranked by ROI
    - Compliance coverage: SOC 2: 78%, ISO 27001: 65%, NIST CSF: 72%

STEP 5: Download the PDF report
─────────────────────────────────
  GET http://localhost:8000/assessments/{run_id}/vendors/V-ACME/report

  → Downloads a cryptographically-anchored PDF/A audit report.
  → This PDF is what you hand to your auditor, CISO, or legal team.

 PART 5 — INTERMEDIATE: MAGIC LINK VENDOR INTAKE
================================================================================

THE SCENARIO: Instead of emailing a CSV template to a vendor and manually
processing their answers, you send them a secure link that opens a web form.
They fill it in, hit Submit, and SENTINEL scores them automatically.

STEP 1: Generate a magic link invitation
─────────────────────────────────────────
  POST http://localhost:8000/invitations/
  Body (JSON):
  {
    "vendor_name": "Acme Cloud Corp",
    "vendor_domain": "acme.com",
    "contact_email": "security@acme.com"
  }

  → Response:
  {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "magic_link": "http://localhost:8000/portal/assess?token=...",
    "vendor_id": "V-ACMECLOU",
    "expires_at": "2026-06-23T14:23:00Z"
  }

STEP 2: Send the magic link to the vendor
──────────────────────────────────────────
  Email the "magic_link" URL to security@acme.com.
  The link is valid for 72 hours and is single-use.

STEP 3: Vendor clicks the link
────────────────────────────────
  The vendor opens the URL in their browser.
  A clean, professional SENTINEL assessment form appears.
  They fill in their answers and hit Submit.
  → SENTINEL automatically runs the full scoring pipeline.
  → Slack/Teams alert fires if they are flagged as High Risk.

WHY THIS BEATS ONETRUST:
  OneTrust requires vendors to create accounts, remember passwords, and
  navigate a complex portal. SENTINEL's magic links require ZERO setup
  from the vendor. Click. Fill. Done. Response rate goes up by 40-60%.

 PART 6 — INTERMEDIATE: AI PDF EVIDENCE VERIFIER
================================================================================

THE SCENARIO: Acme Cloud Corp says "Yes, we have MFA" on their questionnaire.
But their SOC 2 report has a qualified exception for MFA on legacy systems.
SENTINEL's AI catches this contradiction automatically.

STEP 1: Upload the vendor's SOC 2 or ISO 27001 PDF
────────────────────────────────────────────────────
  POST http://localhost:8000/assessments/auto-assess
  Form-data:
    file    = acme_soc2_report.pdf
    controls = MFA,Encryption,Incident Response,Penetration Testing,Backups

  → Response (one object per control):
  [
    {
      "control_name": "MFA",
      "is_present": true,
      "evidence_quote": "...multi-factor authentication is enforced on all production systems...",
      "page_number": 14,
      "confidence_score": 0.85
    },
    {
      "control_name": "Encryption",
      "is_present": false,
      "evidence_quote": "No matching controls located in the uploaded evidence document.",
      "page_number": 0,
      "confidence_score": 0.90
    }
  ]

IMPORTANT NOTES:
  - Works with NO API keys (local regex keyword scanning fallback)
  - Upgrade to AI-grade precision by adding OPENAI_API_KEY to .env
  - Results are cross-referenced against questionnaire answers in the PDF
  - Catches "evidence washing" — vendors who lie on questionnaires

WHY THIS BEATS ONETRUST:
  OneTrust lets vendors upload PDFs but a human analyst must read them.
  SENTINEL reads and extracts controls automatically. 4 hours → 5 minutes.

want to automatically check if any of them have been breached, had their

────────────────────────────────────────────────────────────────────────

────────────────────────────────────────────
  POST http://localhost:8000/monitor/run

  → Response:
  {
    "message": "Continuous monitoring cycle completed.",
    "vendors_checked": 12,
    "results": [
      { "vendor_id": "V-ACME", "avg_external_score": 61.3, "status": "Stable" },

 PART 10 — ENTERPRISE: LIVE WEIGHT TUNING (NO-CODE)
================================================================================

THE SCENARIO: Your legal team says "PCI-DSS compliance is now our top priority
due to a new payment processing contract. Increase the weight of PCI controls."

Without SENTINEL: You'd have to wait for an OneTrust consultant ($300/hr).
With SENTINEL: You update the YAML and it takes 30 seconds.

STEP 1: Retrieve the current weights
──────────────────────────────────────
  Open: rules/weight_config.yaml

STEP 2: Adjust weights via API
────────────────────────────────
  POST http://localhost:8000/schemas/weights
  Body (raw text, YAML format):
  ---
  category_weights:
    CAT_PCI_NETWORK: 0.30      # increased from 0.20
    CAT_PCI_DATA_PROT: 0.30    # increased from 0.20
    CAT_DATA_HANDLING: 0.15
    CAT_ACCESS_CONTROLS: 0.15
    CAT_INCIDENT_RESPONSE: 0.10

  → Response: { "message": "Weights configuration updated successfully." }
  → The NEXT assessment run uses the new weights immediately.
  → The configuration hash in the audit log changes, creating a permanent
    record that weights were updated (cryptographic audit trail).

 PART 12 — TROUBLESHOOTING (Most Common Issues)
================================================================================

ISSUE: "no such table: users" error in tests
  FIX: The test database is not initialised. Run:
    python3 -m pytest -v (uses conftest.py which auto-creates tables)

ISSUE: Webhook alerts not firing
  FIX: Check WEBHOOK_URL and WEBHOOK_PROVIDER in your .env file.
       Test with: curl -X POST http://localhost:8000/monitor/run

ISSUE: BitSight/HIBP returning mock data
  FIX: Add real API keys to .env. Mock data fires when key is "dummy_key" or empty.

ISSUE: PDF report shows "assets not found"
  FIX: Ensure ./assets/fonts/ contains font files AND ./assets/logo.png exists.

ISSUE: Simulation endpoint returns 500
  FIX: Verify questionnaire_schema.yaml and weight_config.yaml exist and are valid.

ISSUE: Magic link says "expired"
  FIX: Links expire in 72 hours. Generate a new invitation with POST /invitations/

ISSUE: Jira webhook not syncing tickets
  FIX: Configure Jira to send webhooks to POST /integrations/jira-webhook
       The issue_key must match the external_key stored in SENTINEL.
